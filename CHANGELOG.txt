# 版本更新日志

## [版本号] - 4.0.2
### 优化
- 优化预约模板和历史记录区域布局，模板名称、模板选择和操作按钮不再互相挤压。
- 前端轮询状态时仅在数据变化后更新对应 DOM，避免当前预约列表卡片反复重绘和跳动。
- 优化 Safari 下 select 和按钮的显示稳定性。

## [版本号] - 4.0.1
### 修复
- 修复 FastAPI 入口在 Python 3.8 环境下因 `tuple[str, str]` 类型标注导致导入失败的问题。

### 优化
- `fastapi_app.py` 新增直接运行入口，可使用 `python fastapi_app.py` 启动本地服务。

## [版本号] - 4.0.0
### 新增
- 新增 FastAPI 主入口 `fastapi_app.py`，将可视化操作迁移为 API + 静态前端。
- 新增 `web/` 前端，采用 Apple Liquid Glass 风格设计，支持玻璃面板、状态动效、实时日志刷新和响应式布局。
- 新前端保留预约列表、预热、开始/停止预约、预约模板、历史记录、当前预约查询取消和账号管理能力。

### 优化
- 新前端不再受 Gradio 组件布局限制，预约队列、模板、账号和日志状态可在同一工作台集中查看。
- 后台预约任务通过 API 启动，前端轮询状态和日志，长时间等待中午开抢时页面不会阻塞。

### 安全与隐私
- API 返回预约列表时不返回真实密码或编码后密码，只返回密码是否已缓存或本次已填写。
- 预约模板和历史记录继续只保存公开预约字段，不保存密码。

## [版本号] - 3.2.0
### 新增
- Gradio 前端新增预约模板功能，可保存当前预约列表并一键加载。
- README 新增预约模板使用说明和本地模板文件说明。

### 安全与隐私
- 预约模板只保存学号、座位、区域和时间，不保存密码。
- 模板文件保存在本地 `cache/templates.json`，继续由 `.gitignore` 忽略。

## [版本号] - 3.1.0
### 新增
- Gradio 前端新增账号管理页面，支持查看已缓存账号、下线指定账号和一键下线全部账号。
- README 新增最近更新和账号管理说明。

### 修复
- 修复切换学号后密码框仍保留上一个账号输入的问题。
- 优化预约记录查询和取消预约接口解析，避免接口返回结构变化时直接抛出不清晰异常。
- 登录失败时返回明确错误信息，便于区分缺少密码、账号密码错误和接口响应异常。

### 安全与隐私
- 多账号预约、查询和取消都按目标学号独立登录，不复用其它账号登录状态。
- 下线账号时同步清理预热上下文，避免内存里保留旧 session/token。
- 继续确保本地账号、预约历史和座位映射缓存不进入 Git 仓库。
